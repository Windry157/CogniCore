## 🛡️ **生存韧性设计原则：极简主义 + 绝对自主**

### 原则 1：不依赖任何外部服务

| 依赖项 | 如果消失了，系统会怎样？ | 我们的设计 |
|--------|--------------------------|------------|
| 网络 | 用户无法访问在线服务 | ✅ 完全离线运行，不需要网络 |
| 云服务 | 数据和计算在云端 | ✅ 所有计算和存储在本地 U 盘 |
| Ollama AI 后端 | AI 功能失效 | ✅ 降级到 System1 规则引擎，没有 AI 也能用 |
| 操作系统更新 | 兼容性问题 | ✅ 兼容 Windows 7+ / Linux / macOS，10 年不更新也能用 |

---

### 原则 2：数据完全主权——"我的数据，只在我手里"

| 数据主权特性 | 说明 |
|--------------|------|
| **100% 本地存储** | SQLite 文件直接在 U 盘，没有任何数据在第三方服务器 |
| **无数据上传** | 系统设计为不上传任何数据到外部（除非用户主动） |
| **完整导出能力** | 所有数据可导出为 JSON/CSV，即使系统不在了，数据依然可读 |
| **物理隔离选项** | U 盘可物理隔离，放在保险箱里，网络攻击永远碰不到 |

---

### 原则 3：极简可重建性——"即使团队全灭，也能重建"

| 重建基础 | 说明 | 重建时间估算 |
|----------|------|--------------|
| **核心代码量** | 核心逻辑 < 10,000 行，任何有经验的 Python 开发者能理解 | 1-2 周读懂代码 |
| **14 份专业文档** | 从架构到运维，从设计到验收，每一步都有文档 | 1 个月完成重建 |
| **模块化设计** | 每个模块独立，可以逐个重建，不需要一次性全量 | 先建核心，再逐步完善 |
| **无技术锁定** | 不依赖特殊框架或云服务，可以在任何 Python 环境运行 | 可移植到任何设备 |
