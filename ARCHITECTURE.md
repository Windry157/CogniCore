## 🔄 3. 数据管道完整流程

### 3.1 端到端数据流

```
[步骤 1] 用户输入
    │
    ▼
[步骤 2] DecisionEvent 封装 (event_id=UUID)
    │
    ▼
[步骤 3] Schema 校验
    │  ├─ 成功 → 继续
    │  └─ 失败 → 记录、返回
    │
    ▼
[步骤 4] 幂等性检查 (event_id 是否已处理?)
    │  ├─ 是 → 跳过、返回成功
    │  └─ 否 → 继续
    │
    ▼
[步骤 5] 发布到消息队列 (内存/Redis)
    │
    ▼
[步骤 6] 消费者接收
    │
    ▼
[步骤 7] 重试循环 (指数退避)
    │  ├─ 尝试 1 → 失败? → 等待 1s → 重试
    │  ├─ 尝试 2 → 失败? → 等待 2s → 重试
    │  ├─ ...
    │  └─ 尝试 N (max_retries) → 失败? → 进入 DLQ
    │
    ▼
[步骤 8] 持久化到 SQLite
    │
    ▼
[步骤 9] 审计日志记录
    │
    ▼
[步骤 10] 指标更新 & 返回成功
```

## 📊 4. 数据库 Schema 设计

### 4.1 核心表

#### users（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | 用户 ID |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |
| metadata | TEXT | 扩展元数据 (JSON) |

---

#### sessions（会话表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT PK | 会话 ID |
| user_id | TEXT FK | 用户 ID |
| created_at | TEXT | 创建时间 |
| last_active | TEXT | 最后活跃时间 |
| state | TEXT | 会话状态 (JSON) |

---

#### decision_events（决策事件表）

| 字段 | 类型 | 说明 |
|------|------|------|
| event_id | TEXT PK | 事件 ID (幂等键) |
| session_id | TEXT FK | 会话 ID |
| input_text | TEXT | 输入文本 |
| output_text | TEXT | 输出文本 |
| confidence | REAL | 置信度 [0, 1] |
| mode | TEXT | 决策模式 (system1/system2/hybrid) |
| metadata | TEXT | 元数据 (JSON) |
| timestamp | TEXT | 事件时间 |
| created_at | TEXT | 创建时间 |

---

#### dead_letter_queue（死信队列表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK AUTOINCREMENT | DLQ 记录 ID |
| event_id | TEXT | 原事件 ID |
| original_message | TEXT | 原始消息 (JSON) |
| error_message | TEXT | 错误信息 |
| retry_count | INTEGER | 已重试次数 |
| status | TEXT | 状态 (pending/resolved) |
| created_at | TEXT | 创建时间 |
| resolved_at | TEXT | 解决时间 |

---

#### audit_log（审计日志表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER PK AUTOINCREMENT | 审计 ID |
| event_type | TEXT | 事件类型 |
| entity_type | TEXT | 实体类型 |
| entity_id | TEXT | 实体 ID |
| before | TEXT | 变更前 (JSON) |
| after | TEXT | 变更后 (JSON) |
| metadata | TEXT | 元数据 (JSON) |
| created_at | TEXT | 创建时间 |

## 🛡️ 6. 容错与恢复机制

### 6.1 快速故障恢复矩阵

| 故障类型 | 影响 | 自动恢复 | 人工干预 |
|----------|------|----------|----------|
| 瞬时网络抖动 | 低 | ✅ 重试机制 | 否 |
| 数据库写入失败 | 中 | ✅ 幂等 + 重试 | 否 |
| 有毒消息 | 中 | ✅ DLQ 隔离 | 修复数据 |
| 进程崩溃 | 高 | ✅ 状态恢复 | 重启 |
| 磁盘满 | 高 | ❌ 否 | 清理空间 |
